회원의 기능
1. 추천 로또 중에 즐겨찾기 추가 (DB에 50개 까지 할당 할 수 있게함)
2. 즐겨찾기한 로또의 통계 데이터 검색 기능
	2-1. 즐겨찾기 한 것 중 과거의 당첨 3등까지 당첨된게 있는지
	2-2. 이전 5주동안 나온 번호에 해당되는 번호가 있는지 없는지
3. 당첨 회차 조회 기능 --> 구현 완

로또와 맴버의 관계

맴버1 --- 로또 다

맴버 id가 pk이고 로또는 fk로 가짐
로또 pk는 로또 id 
저장하면 계속 쌓이는 형식이고
한 회원당 50개 넘으면 가장 나중에 만들었던 것 부터 삭제

컨트롤러에서 북마크 번호, Lotto 형식의 Map을 보내서 저장시 가져온 Map을 순회하면서 저장하면 될듯? LinkedHashMap

로또에 필요한 필드는 맴버id
그리고 북마크 여부 (boolean)


회원 로또 리포지토리 필요 메서드
1. 북마크한 모든 걸 저장하는 메서드 맴버를 따질 필요가 있다!
	맴버로 조인시켜서 개수를 따진 다음 50개가 넘으면 날짜로 가장 나중껄 지우고 새로 넣기
2. 저장한걸 다 꺼내는 메서드 (맴버 아이디 필요)
3. 로또 리스트를 받아 제거하는 메소드 (맴버 아이디 필요)

회원 로또 서비스 필요 메서드
1. 중복 검사한 북마크 된 로또 리스트를 리포지토리에 넘겨주는거
2. 로또를 만들고 최근 8개만 관리하고 컨트롤러에 전달하는 메소드
3. 다 꺼내는 메소드
4. 북마크를 해제한 로또 리스트를 받아 리포지토리에서 제거하는 메소드
